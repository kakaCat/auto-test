## 核心原则

### 文件夹分类隔离
- 使用文件夹对不同功能进行分类
- 每个文件夹职责单一且明确
- 便于代码组织和维护

### TypeScript 最佳实践
- 复杂对象形状首选接口而非类型别名
- 使用类型化对象进行复杂状态管理
- 避免使用 `any` 类型，优先使用 `unknown`
- 使用严格模式 (strict: true)

### Vue 3 组件规范
- 使用 Composition API
- 组件 Props 类型定义
- 事件处理规范
- 生命周期使用规范

### 代码组织
- 按功能模块分类
- 统一的命名规范
- 清晰的目录结构
- 合理的文件拆分

## 使用指南

1. 新项目必须遵循前端编码规范
2. 现有项目逐步重构到新规范
3. 代码审查时严格检查规范遵循情况
4. 定期更新和完善规范内容
