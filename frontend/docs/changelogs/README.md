# 变更记录（占位）

> 说明：遵循 Keep a Changelog 风格进行记录，聚焦用户可感知的变化。当前为占位版本，创建后续按时间顺序追加。

## 记录规范
- 日期格式：`YYYY-MM-DD`
- 分类：Added / Changed / Fixed / Deprecated / Removed / Security
- 每条变更尽量可验证、可回溯，关联 PR/Commit 可选

## 示例
```
## [Unreleased]
### Added
- 初始化文档目录与导航

## [2024-01-15]
### Changed
- 重构 API 文档结构至 frontend/docs/api
```
